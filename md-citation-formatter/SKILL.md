# Md Citation Formatter

## 概述

此技能专门用于规范化学术性或深度研究型 Markdown 文档中的数字引用格式。它能精准识别作为引用的数字，并将其包裹在 `[]` 中，同时通过量词排除逻辑，确保不会误伤文中正常的数值描述。

## 核心功能

### 1. 批量引用规范化
使用内置脚本批量处理文件，将“文字+数字+标点”转换为“文字+[数字]+标点”。

**转换示例：**
- `研究结论13。` -> `研究结论[13]。`
- `特征14，` -> `特征[14]，`
- `储备资产15` -> `储备资产[15]`

**自动避开（不转换）：**
- `15只螃蟹` (后面带量词)
- `13,000台` (千分位/带量词)
- `2026年` (年份)
- `15.5%` (百分比)

## 使用方法

### 批量处理文件
你可以要求 Gemini CLI 对特定文件或整个目录运行规范化脚本：

```bash
node md-citation-formatter/scripts/format_citations.cjs <文件路径>
```

## 识别规则说明

脚本采用以下逻辑确保转换的准确性：
1. **前缀检查**：数字前必须是非数字且非 `[` 的字符（通常是汉字或引用前词）。
2. **数字位宽**：默认处理 1-99 之间的引用编号。
3. **后缀排除**：如果数字后紧跟量词（如 只、个、元、吨、年、% 等）或属于千分位格式，则判定为普通数值，不予转换。
4. **标点锚定**：引用通常出现在句号、逗号或其他停顿标点之前。
